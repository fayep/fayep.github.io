Adventures in Ruby

  • Learning Ruby by writing an adventure game.


  • Lesson 1: Variables and Constants

What is an adventure game?

  • An interactive fiction
  • Text is used to describe the environment
  • Commands are input to interact
  • Usually of the form:
    > GO NORTH
    > OPEN BOX

What is Ruby?

  • An interpreted programming language
  • Created by Yukihiro Matsumoto
  • Everything is an Object
  • Everything can return a value
  • Metaprogramming is possible

Parts of an Adventure Game

  • Rooms
  • Objects
  • Player
  • Keys
  • Doors

What is a room?

  • An identifier
  • A short description
  • A long description
  • Maybe contents
  • Exits

What is an identifier?

  • Unique
  • Easily identified
  • Ruby uses Symbols for this
  • Symbols look like: :Symbol_2

Symbol facts

  • Symbols are immutable
  • Start with a colon
  • Contain letters, numbers and underscores
  • Cannot start with a number
  • May contain other characters
  • Operators are symbols: :+ :- :<<
  • Other special characters: @$ !? "'

About those special cases...

  • @ means "Instance Variable"
  • @@ means "Class Variable"
  • $ means "Global Variable"

  • ! means "Mutate"
  • ? means "Boolean"

Short Description

  • value = 'A String of characters'
  • enclosed by quotes
  • Use single quotes unless
  • formatting = "interpreting #{value}"

Long Description

  • long = <<-endmarker
    This is the content of the string
    More content
    endmarker
  • long2 = %q{This is the content of the string
    More content}

Arrays / Lists

  • a = []
  • b = ['a',2,:charlie]
  • c = 'this can be an array'

Arrays / Lists

  • a.empty?; b[1]
  • c.each_char
  • c.each_byte

Hashes / Structs

  • Store keys and values
  • Things associated with each other
  • Group things
  • teacher = {
      name: "Faye",
      nationality: :welsh,
      status: :married
    }

Blocks

  • Blocks wrap code
  • Blocks return the last value in them
  • begin
      c = 'test'
    end
  • What is the value of c after this?

More Blocks

  • Braces can be used instead of begin/end
  • {c = 'test'}
  • We try to only use braces for single line blocks

Single line blocks?

  • A little can go a long way
  • mylist=['foo','bar','baz']
    mylist.each { |item| puts item}

Iteration

  • ['foo','bar','baz'].each { |item| puts item }
  • (1..10).each { |item| puts item }
  • %i(a b c d).map(&:to_s)

What's that % stuff?

  • Ways to group to make things easier
  • /foo/ == %r{foo}
  • 'foo bar' == %q{foo bar}
  • "foo bar" == %Q{foo bar}
  • [:foo, :bar] == %i{foo bar}
  • ['foo', 'bar'] == %w{foo bar}
  • ["foo", "bar"] == %W{foo bar}

Conditionals

  • How to do either/or
  • if <condition>
      puts 'Condition is true'
    else
      puts 'No can do'
    end

Asking for input

  • yourname = gets
    puts "I think #{yourname} is your name."
  • Includes the "enter" you pressed at the end.
  • The string method "chomp" eats the enter from the end
  • yourname.chomp or gets.chomp work

Case..when..else..end

  • If you're comparing the same variable to lots of things, then you should use case. It has blocks like if..else..end but instead of specifying the entire condition each time, you specify the variable and the matches separately.

Case example

  • puts case gets.chomp
      when 'north' then 'you walk north'
      when 'east' then 'you swim east'
      when 'west' then 'it is dark'
      else "you can't go that way"
    end
