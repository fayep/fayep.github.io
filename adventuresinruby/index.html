What is an adventure game?

What is an identifier?

  • Unique
  • Easily identified
  • Ruby uses Symbols for this
  • Symbols look like: :Symbol_2

Symbol facts

  • Symbols are immutable
  • Start with a colon
  • Contain letters, numbers and underscores
  • Cannot start with a number
  • May contain other characters
  • Operators are symbols: :+ :- :<<
  • Other special characters: @$ !? "'

About those special cases...

  • @ means "Instance Variable"
  • @@ means "Class Variable"
  • $ means "Global Variable"

  • ! means "Mutate"
  • ? means "Boolean"

What's that % stuff?

  • Ways to group to make things easier
  • /foo/ == %r{foo}
  • 'foo bar' == %q{foo bar}

  • ['foo', 'bar'] == %w{foo bar}
  • ["foo", "bar"] == %W{foo bar}

  • If you're comparing the same variable to lots of things, then you should use case. It has blocks like if..else..end but instead of specifying the entire condition each time, you specify the variable and the matches separately.

Example

class IrbView < Adventure::View
  def <<(content)
    case content
    when String
      puts content
    when Array
      content.each do |item|
        puts " #{item}"
      end
    else
      raise "Don't know how to output #{content.class} yet."
    end
  end
end

Inheritance

  • Why was there a < after the classname?
  • If an object is like another we can inherit
  • What does that mean?

Base class for views

module Adventure
  class View
    class << self
      def instance(klass=nil)
        @@instance ||= klass.new
      end

      def <<(content)
        raise "no instance" unless @@instance
        @@instance << content
      end
    end
  end
end

Lots to take in!

  • module Adventure
    • this is to group our classes
  • class View
    • no < means this is a base class.
  • class << self
    • syntax for defining class methods
    • can be called without an instance
    • yes, that means new is a class method

Class methods

  • This technique is often used for logging
  • @@ means the variable is a class attribute
  • it has the same value everywhere.
  • setting with ||= means don't change the value if it has one.
  • class is a reserved word so we spell it oddly

So what does this do?

  • View << "stuff"
  • Delegates the method to our chosen class instance
  • Gives us a general way to do something we can be more specific about later
  • This is a simple "abstraction"
